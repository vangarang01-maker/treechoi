# 02. 백엔드 및 기능 로직 개선 (Backend & Logic)

**마지막 수정일**: 2026-03-15

본 문서는 `sbe-jira-ui` 프로젝트의 서버 측 API, 데이터 처리 로직 및 인프라 개선 사항을 정의합니다.

---

## 1. 핵심 기능 로직 (AI & Data)

### 💡 [BE-1] 자연어 의도 분석 에이전트 (Agent Query)
- **개요**: 사용자의 자연어 입력을 분석하여 SEARCH, ACTION, CHAT 중 의도를 판단하고 파라미터를 추출합니다.
- **구현**: `lib/gemini.py` -> `api_gemini_process_agent()`
- **주요 로직**:
  - JQL 자동 생성 (작업 범위, 프로젝트, 이슈 타입 등 반영)
  - 액션 파라미터 추출 (이슈키, 담당자 사번, 상태명, 댓글 내용 등)
- **상태**: ✅ 구현 완료

### 💡 [BE-2] 유사 이슈 임베딩 및 검색
- **개요**: 완료된 이슈들의 제목과 내용을 벡터화하여 현재 이슈와 가장 유사한 건을 추천합니다.
- **구현**: `lib/embedding.py`
- **주요 로직**:
  - `SentenceTransformer` 기반 한국어 임베딩
  - `data/` 디렉토리에 사번별 캐시(JSON/NP) 구축
  - 코사인 유사도 기반 Top 3 이슈 추출
- **상태**: ✅ 구현 완료

### 💡 [BE-3] Gemini 기반 처리 초안 생성
- **개요**: 유사 완료 이슈의 처리 방식을 참고하여 현재 이슈의 해결 댓글 초안을 자동 생성합니다.
- **구현**: `lib/gemini.py` -> `api_draft_comment()`
- **주요 로직**:
  - 현재 이슈 + 유사 이슈 상세를 조합하여 퓨샷(Few-shot) 프롬프트 구성
  - 실무 중심의 간결한 문체로 초안 생성
- **상태**: ✅ 구현 완료

## 2. API 엔드포인트 설계 (`config_ui.py`)

| 경로 | 메서드 | 설명 |
|------|--------|------|
| `/api/jira-check` | POST | Jira PAT Token 유효성 검증 및 사용자명 조회 |
| `/api/jira-update` | POST | 이슈에 댓글 추가 또는 상태 변경 실행 |
| `/api/jira-transitions` | GET | 특정 이슈가 전이 가능한 상태 목록 조회 |
| `/api/gemini-chat` | POST | 대화 히스토리를 포함한 Gemini 채팅 응답 |
| `/api/agent-query` | POST | 자연어 입력에 대한 의도 분석 및 JQL 생성 |
| `/api/ai-verify` | POST | Top-N 유사 이슈 중 Gemini가 최적 1건 선별 |
| `/api/draft-comment` | POST | 최적 매칭 이슈 참고 처리 초안 생성 |
| `/api/similar-issues` | POST | 미해결 이슈 임베딩 후 유사 완료 이슈 추천 |
| `/api/embedding-build-stream` | GET (SSE) | 임베딩 캐시 구축 진행 상황 실시간 스트리밍 |

## 3. 인프라 및 배포 (Infrastructure)

### 💡 [INF-1] 멀티유저 대응 서버 설계
- **개요**: 서버가 API 키를 저장하지 않는 '무상태(Stateless)' 아키텍처로 전환했습니다.
- **핵심**: 모든 API 요청 바디 또는 헤더에 클라이언트가 보유한 `env` (localStorage 데이터)를 포함하여 전달합니다.
- **장점**: 동일 컨테이너에서 여러 사용자가 각자의 계정으로 동시 사용 가능.
- **상태**: ✅ 구현 완료

### 💡 [INF-2] 컨테이너 기반 배포 (Docker)
- **개요**: 환경에 구애받지 않는 실행을 위해 Docker 대응을 완료했습니다.
- **구성**:
  - `Dockerfile`: `python:3.11-slim` 기반 경량 이미지
  - `docker-compose.yml`: `data/` 볼륨 마운트로 컨테이너 재생성 시에도 임베딩 캐시 보존
- **상태**: ✅ 구현 완료

### 💡 [INF-3] 실시간 상태 피드백 (SSE)
- **개요**: 긴 시간이 소요되는 임베딩 작업을 위해 SSE(Server-Sent Events) 프로토콜을 구현했습니다.
- **상태**: ✅ 구현 완료
