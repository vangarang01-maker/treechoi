**작성일**: 2026-03-15

## 개요

이슈 키를 입력하면 Jira 이슈의 현재 상태를 자동 감지하고, 해당 단계에서 작성해야 할 필드의 AI 초안을 생성해 주는 보조 도구.
생성된 초안은 클립보드로 복사해 사용자가 Jira에 직접 붙여넣는 방식.

**대상**: 변경관리(일반/긴급) + 서비스요청관리(4가지 업무유형)

---

## 확정된 설계 결정사항

| 항목 | 결정 |
|------|------|
| 출력 방식 | 클립보드 복사 — Jira에 API로 직접 저장하지 않음 |
| 결재상신 | 자동화 안 함 — 사용자가 Jira에서 직접 |
| UI 위치 | 신규 탭 추가 ("🪄 처리 마법사" 또는 유사 명칭) |
| 대상 이슈 유형 | 변경관리 + 서비스요청관리 모두 포함 |
| 구현 방식 | 단계별 — Phase 1부터 테스트 가능한 단위로 순차 구현 |

## 변경관리

### 처리 흐름

**일반 변경관리:**
```
미해결 → 영향도분석 → [변경계획결재①] → 개발중 → 배포계획수립 → [현업확인결재②] → 현업확인 → [배포계획결재③] → 배포 → 배포완료
```

**긴급 변경관리:**
```
미해결 → 영향도분석 → [긴급배포확인결재①] → 긴급배포 → 배포
```

**긴급 판단 기준**: 이슈 제목에 `[긴급]` 또는 `(긴급)` 포함 여부 (우선순위 필드 아님)

### 변경 유형 (`customfield_16460`)

| 대분류 | 세부 유형 |
|--------|---------|
| 프로그램 | `[프로그램] UI/서비스/기능` |
| | `[프로그램] 배치` |
| | `[프로그램] I/F` |
| DB | `[DB] 데이터변경(DML)` |
| | `[DB] 테이블변경` |
| *(제외)* | `[단순변경]` — 사용 빈도 낮아 제외 |

### 단계별 AI 초안 생성

> 커스텀 필드 전체 레퍼런스: `D:/projects/02.JIRA/변경관리-유형별처리.md`

> 현재 Jira 상태를 감지하여 해당 단계에 필요한 필드만 표시. 앞 단계 내용이 뒷 단계 초안에 반영됨.

| Jira 단계 | 마법사가 할 일 | AI 역할 | 사람 역할 |
|-----------|-------------|---------|---------|
| 미해결 | Pre-flight check | `customfield_16460` 자동 감지, 긴급 여부 제목에서 감지 | 변경 유형 확인/수정, 긴급 override |
| | 요건정의서 초안 | `description` → 요건정의서 테이블(번호/구분/요구사항/검증결과) 변환 | 검토/수정 후 복사 |
| 영향도분석 중 | 변경검토회의 초안 | 이슈 내용 기반 5섹션 생성 (영향범위/타시스템영향도/영향분석결과/검토회의결과/변경내용) | 회의일시·참석자 직접 입력, 전체 검토 후 복사 |
| 변경계획결재① 전 | 결재라인 미리보기 | 사용자 프리셋에서 불러오기 | 확인 후 Jira에서 직접 상신 |
| 개발중 | 테스트케이스 초안 | 변경검토회의 내용 참고. 프로그램: 정상4+비정상2 / DB: 1건 | 내용 수정 후 복사 |
| 배포계획수립 중 | 배포절차 초안 | 변경내용 요약 + 서버/절차/일시 + 체크리스트(선배포/대고객/트래픽/사전테스트/백업/롤백/고객사공유/현업검증) | 서버 IP·배포일시 직접 입력, 검토 후 복사 |
| | 원복계획 초안 | 배포절차 기반 원복 방법 | 검토/수정 후 복사 |
| 현업확인결재② 전 | 결재라인 미리보기 | 사용자 프리셋에서 불러오기 | 확인 후 Jira에서 직접 상신 |
| 배포계획결재③ 전 | 결재라인 미리보기 | 사용자 프리셋에서 불러오기 | 확인 후 Jira에서 직접 상신 |
| 긴급배포확인결재① 전 | 결재라인 미리보기 | 사용자 프리셋에서 불러오기 | 확인 후 Jira에서 직접 상신 |

### 결재라인 프리셋

> 사람이 바뀔 수 있으므로 자동 구성 대신, 환경설정에서 사용자가 직접 등록해두고 재사용.
> `localStorage`에 저장, 처리 마법사에서 불러와 미리보기로 표시.

| 프리셋 항목 | 해당 Jira 필드 |
|------------|--------------|
| 변경계획결재 라인 (일반) | `10733~10809` 체인 |
| 현업확인결재 라인 (일반) | `11500`, `15018` |
| 배포계획결재 라인 (일반) | 추가 조사 필요 |
| 긴급배포확인결재 라인 | `11500`, `15018`, `16007` |

## 서비스요청관리

### 처리 흐름

```
미해결 → 처리중 → 완료
  (결재 단계 없음 — 담당자가 직접 처리 후 완료 전환)
```

### 업무유형 감지

`업무유형` 필드(customfield 번호 추가 조사 필요) 값을 읽거나, 감지 불가 시 사용자가 직접 선택.

### 유형별 양식 및 AI 자동화 수준

**1) 계정/권한 처리**
- 필수 항목: 요청내용, 대상시스템, 대상자, 사번, 사유
- AI 수준: 🟢 높음 — `description` 내용에서 항목 추출 가능
- 비고: EP MCP(`t_ep_generateForm`) 기존 활용 가능

**2) 데이터추출**
- 필수 항목: 대상시스템, 추출내용, 사유, 개인정보포함여부, SQL, 결과파일
- AI 수준: 🟠 낮음 — SQL과 결과파일은 수동 입력 필수
- 비고: SQL은 AI가 초안 제안은 가능하나 검증 책임은 사람에게

**3) 공통코드 단순변경**
- 필수 항목: 대상시스템, 작업내용, 영향도, 작업절차(정상/비정상), 확인방안, 이슈대응방안
- AI 수준: 🟡 보통 — 작업절차·확인방안 AI 초안 생성 가능

**4) 기타(단순)**
- 필수 항목: 대상시스템, 요청내용, 요청사유, 기타특이사항
- AI 수준: 🟢 높음 — `description` 내용 그대로 정리

## UI 구성

### 화면 흐름

```
[이슈 키 입력] → [조회]
  ↓
이슈 유형 자동 감지
  ├─ 변경관리   → 현재 Jira 상태 감지 → 해당 단계 초안 화면
  └─ 서비스요청관리 → 업무유형 감지/선택 → 해당 유형 양식 화면
  ↓
Pre-flight check (변경관리만)
  - 변경 유형 드롭다운 (현재값 표시, 수정 가능)
  - 긴급 여부 체크박스 (제목 자동감지 + 수동 override)
  ↓
[AI 초안 생성] 버튼 (필드별 개별)
  - 생성된 텍스트는 인라인 편집 가능한 textarea로 표시
  ↓
[복사] 버튼 — 필드별 또는 전체 한 번에 복사
```

## 단계별 구현 계획

| Phase | 구현 내용 | 테스트 방법 |
|-------|---------|-----------|
| 1 | 이슈 키 입력 → 유형/상태 감지 → 다음 할 일 안내 (AI 없음) | 다양한 상태의 이슈 키로 올바른 안내 문구 확인 |
| 2 | 변경관리 Pre-flight check UI (변경유형/긴급여부 확인 화면) | 잘못된 유형 이슈에서 수정 가능한지 확인 |
| 3 | 요건정의서 AI 초안 생성 + 복사 | description → 요건정의서 테이블 변환 품질 확인 |
| 4 | 변경검토회의 AI 초안 생성 + 복사 | 5섹션 구조 + 내용 품질 확인 |
| 5 | 테스트케이스 / 배포절차 / 원복계획 AI 초안 생성 | 각 필드 품질 및 형식 확인 |
| 6 | 서비스요청관리 유형별 양식 초안 생성 | 4가지 유형별 초안 품질 확인 |
| 7 | 결재라인 프리셋 설정 화면 (환경설정 탭 추가) | 프리셋 저장/불러오기 정상 동작 확인 |
