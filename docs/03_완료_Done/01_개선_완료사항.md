# 01. 개선 완료 사항 (Done / Release Notes)

**최종 업데이트**: 2026-03-20

기획, 개발 단계(Sprint)를 거쳐 사용자 환경에 정규 배포된 기능 및 버그/보안 패치의 이력(Release Note)입니다.

---

## 🎉 배포 완료: 사용자 경험 (UX) 최적화
- **온보딩 가이드 접근성 강화**: 첫 진입자를 위해 API 발급 가이드 링크를 카드 전면에 표시하도록 수정.
- **Jira 연결 상태 확인 체계**: PAT Token 유효성을 `⚡ 연결 확인` 버튼 하나로 검증할 수 있도록 툴팁과 함께 개선.
- **대화 및 설정 영속성 유지**: `localStorage` 연계를 통해 새로고침 후에도 기존 업무 맥락이 소멸하지 않도록 조치.
- **Jira 인라인 액션 패널**: `💬 댓글` / `🔄 상태` 등 자주 쓰는 컨트롤을 외부 페이지로 이동하지 않고도 검색 목록에서 직접 조작할 수 있도록 구현.
- **유사 이슈 결과 타입별 그룹핑**: 결과물들을 단순히 섞어서 나열하지 않고 `issuetype` 단위별 섹션으로 구분 및 아코디언 컴포넌트 처리.

---

## 🎉 배포 완료: Gemini AI 프롬프트 (Core AI)
- **자연어 JQL 통합 검색 (Agent Query)**: "최근 담당한 에러 찾아줘"처럼 자연어 질의를 JQL 검색문맥으로 투명하게 변환하여 수행.
- **이슈 처리 마법사 (Workflow Wizard)**: 변경 단계에 맞춰 6종 (배포절차, 결제멘트 등) 양식을 능동적으로 제안 및 초안 자동 작성 지원.
- **맥락 주입형 자동 처리 초안 (Draft Generation)**: 과거 높은 유사도 티켓의 결과(Comment)를 LLM 퓨샷으로 주입시켜 즉시 사용에 무방한 현재 이슈 해결 대댓글 초안 생성 체계 수립.
- **캐시 실시간 진행 현황 통신 (SSE)**: 수백 건의 임베딩 로드 중 시스템 행(Hang) 인식을 피하기 위해 HTTP 통신을 활용하여 실시간 프로그레스 바 스트리밍 적용 완료.
- **프롬프트 분리화 아키텍처**: 백엔드 소스에 매립된 AI 프롬프트 문자열들을 `prompts/` 외부 디렉토리에 10개 별도 파일로 분리. 이제 서버 재시동 없이 프롬프트 개선 가능.

## 🎉 배포 완료: 인프라 버그 및 보안 조치
- **Docker 생태계 환경 지원**: 개발팀 전체가 `docker-compose.yml` 리소스로 일관된 배포 지원 확보.
- **평문 유출 버그 조치 [SEC-1]**: `/api/config` 호출 시 `GEMINI_API_KEY`, `JIRA_PAT_TOKEN` 등의 민감키가 JSON Body에 유출되던 버그 진단 및 `"__set__"` 치환으로 마스킹 완전 수정.
- **403 에러 핸들링 보완 [BUG-2]**: Gemini API의 HTTP 403 장애 발생 시 메시지가 날것으로 화면에 노출되던 문제를 친화적 안내로 파싱.
- **임베딩 캐시 SQLite 전환 [BR-3]**: `embedding_cache.json` → `embedding_cache.db` (SQLite) 전환 완료. `threading.Lock()` 동시 쓰기 잠금 병행 적용. `./data:/app/data` 볼륨 마운트로 컨테이너 재생성 후에도 캐시 보존.
- **멀티스레드 서버 전환 [P0-1]**: `HTTPServer` → `ThreadingHTTPServer` 교체로 동시 요청 처리 가능.

## 🎉 배포 완료: 유사 이슈 검색 UX 개선

- **이슈 키 Jira 링크 연결**: 유사 이슈 검색 결과 카드의 모든 이슈 키(미해결 이슈 키, 유사 이슈 키)를 클릭하면 Jira 원본 페이지로 바로 이동.
- **캐시 구축 유형별 분리**: 기존 단일 "캐시 구축" 버튼을 "🔄 서비스요청관리" / "🔄 변경관리" 2개로 분리. 유형별 독립 구축·갱신 가능, 한 유형 구축 시 나머지 유형 캐시는 그대로 유지.
- **대상 사용자 자유 지정**: 기존에 "나(사번)"가 자동 포함되던 방식을 제거하고 최대 3명을 자유롭게 추가/삭제할 수 있도록 변경.
- **볼륨 마운트 확장**: `ui/`, `lib/` 디렉토리를 Docker 볼륨으로 마운트하여 코드 수정 후 `docker compose restart`만으로 반영 가능 (빌드 불필요).
- **라이트/다크 모드 토글 [5-B]**: 사이드바 하단 버튼으로 테마 전환. CSS `[data-theme]` 변수 기반으로 전체 UI 색상 전환, `localStorage`에 저장되어 새로고침 후에도 유지.
- **앱 이름 변경**: `sbe-jira-mcp` → `스벅 Jira 도우미` (사이드바 브랜드명 + 브라우저 탭 타이틀).

## 🎉 배포 완료: 처리 마법사 — 서비스요청관리 지원

- **서비스요청관리 초안 생성 추가**: 마법사에서 서비스요청관리 이슈 조회 시 업무유형 선택 드롭다운 표시 및 "처리 내용 초안" AI 생성 버튼 제공.
- **업무유형별 프롬프트 4종 추가**: 계정/권한 처리 / 데이터추출 / 공통코드 단순변경 / 기타 — 각 유형에 맞는 항목 자동 추출.
- **업무유형 자동 감지**: Jira customfield에서 업무유형 자동 감지 시도, 실패 시 드롭다운에서 수동 선택.
- **완료 제외 전 상태에서 초안 버튼 노출**: 미해결·처리중·반려·중단 등 완료가 아닌 모든 상태에서 초안 생성 가능.
