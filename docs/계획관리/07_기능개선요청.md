### [UI-E] 유사 이슈 검색 결과 유형별 그룹화
- **요청일**: 2026-03-15
- **배경**: 유사 이슈 검색 결과에서 `서비스요청관리`와 `변경관리` 카드가 섞여 출력되어 식별이 어렵다는 사용자 피드백
- **개선 내용**:
  - 검색 결과를 `issuetype` 기준으로 그룹핑하여 섹션별로 표시
  - 각 섹션 상단에 이슈 유형명과 건수 헤더 표시 (예: `서비스요청관리 (3건)`)
  - 섹션 헤더 클릭 시 해당 그룹 접기/펼치기 토글 지원
  - 백엔드 변경 없음 — 프론트엔드 렌더링 로직만 수정
- **상태**: ✅ 구현 완료 (2026-03-15)

## 대기 중인 요청

### [BR-2] 이슈 처리 마법사 (Processing Wizard) — 신규 기능
- **요청일**: 2026-03-15
- **상태**: 🆕 검토 중
- **배경**: 변경관리/서비스요청관리 이슈를 처리할 때마다 유형별로 다른 양식을 직접 작성해야 해서 반복 공수가 큼
- **원하는 것**:
  - 이슈 키를 입력하면 현재 단계를 자동 감지
  - 해당 단계에서 필요한 필드의 AI 초안을 생성해 주고
  - 사용자가 검토/수정 후 클립보드에 복사해서 Jira에 직접 붙여넣기
  - 변경관리(일반/긴급) + 서비스요청관리(4가지 업무유형) 모두 대상
  - 결재상신은 자동화하지 않음 — 사람이 Jira에서 직접
- **상세 기능 명세**: [기능명세/04_이슈처리마법사_기능명세.md](../기능명세/04_이슈처리마법사_기능명세.md)
