## 전체 흐름

```
[1단계] 설정
  - 나 + 추가 사용자 1~2명 사번 입력 (UI)

[2단계] 캐시 구축 (최초 1회 + 주기적 갱신)
  - 지정한 사용자들의 완료 이슈 조회 (이슈 타입별)
  - 각 이슈 summary+description → text-embedding-004 → 벡터
  - 벡터 파일로 저장 (embedding_cache.json)

[3단계] 유사 이슈 검색
  - 나의 미해결 이슈 조회
  - 각 미해결 이슈 → 임베딩 → 같은 타입의 완료 이슈 벡터와 코사인 유사도
  - 유사도 Top 3 반환

[4단계] 결과 표시
  - 미해결 이슈 카드 + "유사 완료 이슈 Top 3" 펼치기
```

## 상세 설계

### Jira 쿼리

**미해결 이슈 조회 (검색 대상)**
```
assignee in (currentUser(), "사번2", "사번3")
AND status not in ("완료", "반려", "중단")
ORDER BY updated DESC
```

**완료 이슈 조회 (유사도 비교 풀)**
```
assignee in (currentUser(), "사번2", "사번3")
AND status = "완료"
AND issuetype = "{타입}"   ← 타입별로 분리 호출
ORDER BY updated DESC
MAXRESULTS 200             ← 너무 많으면 캐시 구축 시간 오래 걸림
```

### 임베딩 텍스트

**핵심 원칙: 미해결 이슈와 완료 이슈를 동일한 텍스트 구조로 임베딩 → 비대칭 해소**

```
[이슈타입] {summary}
{최초 description 앞 300자}
```

| 이슈 | 임베딩에 포함 | 임베딩에서 제외 |
|------|-------------|---------------|
| 미해결 | summary + 최초 description | - |
| 완료 | summary + 최초 description | 처리과정, 댓글, 결과 |

→ 양쪽 모두 "요청 당시 텍스트"만 사용해서 공평한 비교

### 캐시 파일 구조

`embedding_cache.json`:


### 코사인 유사도 계산

```python
import math

def cosine_similarity(a, b):
    dot = sum(x*y for x, y in zip(a, b))
    norm_a = math.sqrt(sum(x**2 for x in a))
    norm_b = math.sqrt(sum(x**2 for x in b))
    return dot / (norm_a * norm_b)
```
→ numpy 없이 순수 Python으로 구현 (의존성 최소화)

## 서버 구현 (config_ui.py 추가 함수)

### 신규 API 엔드포인트

| 메서드 | 경로 | 기능 |
|--------|------|------|
| GET | `/api/embedding-cache-status` | 캐시 현황 조회 (건수, 갱신일) |
| POST | `/api/embedding-build` | 캐시 구축 / 갱신 (`{"users": ["223733", "사번2"]}`) |
| POST | `/api/similar-issues` | 유사 이슈 검색 (`{"users": [...]}`) |

### `api_embedding_build(users)` 흐름

```python
1. 이슈타입 목록: ["서비스요청관리", "변경관리"]
2. 타입별로 완료 이슈 Jira 조회 (최대 200건씩)
3. 각 이슈 텍스트 구성 (summary + description[:300])
4. Gemini text-embedding-004 호출 (배치로 처리)
5. embedding_cache.json 저장
```

### `api_similar_issues(users)` 흐름

```python
1. 미해결 이슈 조회 (assignee in users)
2. 각 미해결 이슈 임베딩 (Gemini API)
3. 같은 이슈타입의 캐시 벡터와 코사인 유사도 계산
4. Top 3 반환
```

## AI검증 기능

> 요청일: 2026-03-07
> 상태: 🔲 미구현

### 개요

Embedding Top 3는 벡터 유사도 기반이라 **표면적 유사도**만 측정.
AI검증은 Gemini가 실제 이슈 내용을 읽고 **의미적으로 가장 유사한 1건**을 추려내는 2단계 검증.

```
[1단계] Embedding → Top 3 후보 선정 (빠름, 토큰 0)
[2단계] AI검증 → Gemini가 Top 3 읽고 최적 1건 선택 (느림, 토큰 소모)
```

---

| 항목 | 미해결 이슈 | 완료 이슈 (Top 3) |
|------|------------|-------------------|
| 이슈 키 | ✅ | ✅ |
| 이슈타입 | ✅ | ✅ |
| 제목(summary) | ✅ | ✅ |
| 설명(description) | ✅ 전문 | ✅ 전문 |

→ Embedding과 달리 **완료 이슈의 처리 결과까지 포함** → 더 정확한 비교 가능

| 메서드 | 경로 | 입력 | 기능 |
