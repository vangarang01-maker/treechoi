## 3. Jira 설정을 Jira 테스트 탭으로 이동

> 요청일: 2026-03-06

### 요청 내용

Jira 설정(BASE_URL, PAT Token, 사용자명)을 💬 Jira 테스트 탭 내에서 관리.

### 결과 구조

1번(Gemini 통합) + 3번(Jira 이동) 완료 시 탭 구성:

| 탭 | 내용 |
|----|------|
| 🤖 Gemini | Gemini 설정(접이식) + 채팅 |
| 💬 Jira 테스트 | Jira 설정(접이식) + Jira 쿼리 채팅 |

→ **⚙ 설정 탭 완전 제거** (Gemini/Jira 설정이 각 탭으로 분산)

### 구현 방식

Gemini 탭(1번)과 동일한 접이식 패턴 적용:
- 💬 Jira 테스트 탭 상단에 `[⚙ Jira 설정 ▶]` 토글
- 펼치면 BASE_URL, PAT Token(눈 아이콘), 사용자명 입력 + 저장 버튼
- Jira 설정이 비어있으면 자동 펼침

### 변경 파일

| 파일 | 변경 내용 |
|------|-----------|
| `ui/index.html` | Jira 탭에 접이식 설정 패널 추가, ⚙ 설정 탭 전체 제거, 탭 버튼 2개로 축소 |
| `ui/script.js` | Jira 탭 토글/저장 함수, BASE_URL 미설정 시 자동 펼침 |
| `ui/style.css` | 추가 스타일 없음 (Gemini와 동일 클래스 재사용) |

## 요청 내용

## 구현 방식

### HTML

`check-row` 안에 물음표 아이콘 + 툴팁 마크업 추가.

```html
<div class="check-row">
  <div class="tooltip-wrap">
    <span class="help-icon">?</span>
    <div class="tooltip-box">
      <b>상태 확인 판단 기준</b><br>
      countTokens API로 ping 전송<br>
      🟢 정상 — API 키 유효, 모델 존재<br>
      🟡 할당량 초과 (429) — quota 소진<br>
      🔴 모델 없음 (404) — 모델명 오류<br>
      🔴 API 키 오류 (400/401/403)<br>
      🔴 오류 — 네트워크 오류 등
    </div>
  </div>
  <button ... >⚡ 상태 확인</button>
  ...
</div>
```

### CSS

- `.help-icon`: 작은 원형 `?` 버튼 (16px, 흐린 색)
- `.tooltip-box`: 기본 `display:none`, `.tooltip-wrap:hover` 시 나타남
- 툴팁 위치: 위쪽 (bottom: 100%) 또는 오른쪽, 화살표 포함
- z-index로 다른 요소 위에 표시

## 5. Gemini 기반 Jira 이슈 검색기

> 요청일: 2026-03-06

### 요청 내용

자연어로 Jira 이슈를 검색할 수 있게 한다.
Gemini가 자연어를 JQL로 변환하고, 실제 Jira에서 검색해 결과를 보여준다.

---

### 현재 구조의 한계

현재 💬 Jira 테스트 탭은 이슈키 / JQL 직접 입력만 가능.
→ JQL 문법을 모르는 사람은 쓰기 어렵다.

---

### 구현 방향 후보

#### 방안 A — Gemini + Jira 통합 채팅 (추천)

🤖 Gemini 탭에서 Jira 검색까지 가능한 **에이전트 방식**.

```
사용자: "내가 담당한 EP 이슈 중 미해결 건 알려줘"
Gemini: (내부적으로 JQL 생성 + Jira 검색 실행)
      → "미해결 EP 이슈 2건이 있습니다: ..."  [이슈 카드 표시]
```

**동작 흐름:**
1. 서버에서 Gemini에게 시스템 프롬프트 전달:
   - "Jira 검색이 필요하면 응답에 `JIRA_SEARCH:{"jql":"..."}` 를 포함하세요"
2. Gemini 응답에서 `JIRA_SEARCH:{...}` 패턴 감지
3. 패턴 발견 시 → Jira API 호출 → 결과를 Gemini에게 2차 전달 → 자연어 최종 답변
4. 패턴 없으면 → 일반 Gemini 응답 그대로 표시

**장점:** 자연스러운 대화 흐름, 이슈 설명·요약도 가능
**단점:** 서버에서 최대 2회 API 호출 (Gemini → Jira → Gemini), 응답이 다소 느림

```
[🤖 Gemini 탭]
사용자: "변경관리 이슈 중 아직 배포 안 된 거 있어?"
AI:     Jira를 검색해볼게요...
        [SCM3-15200] DB 마이그레이션  🟡 영향도분석
        [SCM3-15188] API 배포          🟡 영향도분석
        총 2건입니다. 배포 일정을 잡으시겠어요?
```

---

#### 방안 B — 💬 Jira 탭에 AI 모드 토글

기존 Jira 테스트 탭에 **AI 모드 버튼** 추가.
AI 모드 ON → 자연어 입력 → Gemini JQL 변환 → 검색
AI 모드 OFF → 기존 JQL 직접 입력 방식 유지

```
[💬 Jira 테스트 탭]
[🤖 AI 모드] ← 토글 버튼

AI 모드 ON 시:
  입력창: "이번 달 완료된 변경관리"
  → Gemini가 변환한 JQL: issuetype = "변경관리" AND status = "완료" AND ...
  → Jira 검색 결과 표시
```

**장점:** 탭 분리 명확, JQL도 보여줘서 학습 효과 있음
**단점:** 탭을 넘나들며 AI 대화는 불가능

---

### 시스템 프롬프트 (방안 A 기준)

Gemini에게 전달할 시스템 프롬프트 초안:

```
당신은 Jira 이슈 관리 도우미입니다.
프로젝트: SCM3 (Shinsegae 사내 Jira)
현재 사용자: {JIRA_USERNAME}

Jira 검색이 필요하면 응답에 아래 마커를 포함하세요 (마커는 한 줄에 단독으로):
JIRA_SEARCH:{"jql":"JQL 쿼리","fields":["summary","status","assignee"]}

마커 없이 검색 결과가 전달되면 결과를 자연어로 설명하세요.
JQL에서 현재 사용자는 currentUser() 를 사용하세요.
이슈타입: 변경관리, 서비스요청관리(EP포함), Task, Bug
상태: 미해결, 진행 중, 완료, 반려, 중단, 영향도분석, 배포완료
```
