# 02. 개선 기획 — 기능 명세

**작성일**: 2026-03-14

---

## 개선 원칙

1. **사용자 흐름 중심**: 기술 구조보다 사용자가 자연스럽게 작업을 완료할 수 있는 흐름 우선
2. **점진적 개선**: 현재 Python stdlib 구조를 유지하면서 단계적으로 기능 추가
3. **Zero Configuration 지향**: 처음 실행해도 막힘 없이 사용 가능해야 함

---

## 개선 영역 1: 온보딩 & 환경설정

### 💡 1-A. 초기 설정 마법사 (신규)

**배경**: 새 사용자가 토큰을 어디서 발급하는지 몰라 첫 실행에서 막힘

**개선 내용**:
- 설정 화면에 각 필드별 **"발급 방법" 툴팁 또는 링크** 추가
  - Gemini: `https://aistudio.google.com/app/apikey` 링크 + 간단한 설명
  - Jira: `https://jira.sinc.co.kr/profile` → 개인 액세스 토큰 경로 안내
- `.env` 미존재 시 **설정 탭을 자동으로 열고** "설정이 필요합니다" 배너 노출

### 💡 1-B. Jira 연결 상태 확인 (신규)

**배경**: Gemini와 달리 Jira는 토큰 저장 후 유효성 확인 수단이 없음

**개선 내용**:
- Gemini `⚡ 상태 확인` 버튼처럼 Jira에도 **`⚡ 연결 확인`** 버튼 추가
- 내부적으로 `/rest/api/2/myself` API 호출, 응답에서 사용자명 파싱
- 성공: 🟢 `홍길동 (223733) 연결됨`
- 실패: 🔴 `인증 오류 — 토큰을 다시 확인해주세요 (401)`

## 개선 영역 2: Gemini 채팅 강화

### 💡 2-A. 대화 히스토리 영속화 (중요)

**배경**: 새로고침 시 대화 내용 전부 사라짐

**개선 내용**:
- `localStorage`에 대화 내용 자동 저장 (클라이언트 측, 서버 무관)
- 최대 50턴 보관, 오래된 대화는 자동 삭제
- UI에 **`[🗑 대화 초기화]`** 버튼 제공

### 💡 2-B. 스트리밍 응답 (중요)

**배경**: 긴 응답 시 응답이 올때까지 수초간 빈 화면

**개선 내용**:
- Gemini `streamGenerateContent` API 전환
- 서버에서 SSE(Server-Sent Events)로 토큰 단위 전달
- 사용자에게 타이핑 효과로 즉각적인 피드백 제공

### 💡 2-C. 맥락 주입형 Jira 채팅 (핵심 신규)

**배경**: "이 이슈 요약해줘"처럼 Jira 데이터를 채팅에서 바로 활용 불가

**개선 내용**:
- 채팅창에 **`@SCM3-15200`** 형식으로 이슈 키를 태그하면 자동으로 이슈 내용을 컨텍스트에 주입
- Gemini에게 "이슈 내용 + 사용자 질문"을 함께 전달
- 예시 사용 흐름:
  ```
  사용자: @SCM3-15200 이 이슈 처리 방법을 초안으로 작성해줘
  AI: [SCM3-15200 EP 권한 신청 이슈 기준]
      1. 결재양식 확인 후 ...
  ```

### 💡 2-D. 시스템 프롬프트 커스터마이징 (보조)

**배경**: 내 역할/팀 정보를 매번 채팅에 설명해야 함

**개선 내용**:
- 환경설정에 **"AI 역할 설정"** 텍스트 필드 추가
- 예: `"당신은 사내 IT 서비스 담당자를 돕는 업무 보조 AI입니다. 프로젝트: SCM3, 팀: 운영팀"`
- `generationConfig`의 `systemInstruction` 필드에 자동 삽입

## 개선 영역 3: Jira 조회 & 관리

### 💡 3-A. 자연어 검색 통합 (핵심)

**배경**: JQL 문법을 모르는 사용자는 기존 3개 숏컷 외에는 사용 어려움

**개선 내용**:
- 검색창 상단에 **자연어 입력 모드** 토글 추가
- 자연어 입력 시 `api_gemini_process_agent()`로 JQL 자동 생성
- 생성된 JQL을 UI에 미리 보여주고 (투명한 동작), 사용자가 수정 후 실행 가능
- 예시:
  ```
  입력: "이번달에 완료한 변경관리 이슈 보여줘"
  생성: project = SCM3 AND issuetype = "변경관리" AND status = "완료" 
        AND resolved >= startOfMonth() ORDER BY resolved DESC
  ```

### 💡 3-B. 숏컷 관리 기능 (보조)

**배경**: JQL 숏컷이 코드에 하드코딩되어 수정하려면 소스 편집 필요

**개선 내용**:
- UI에서 숏컷 추가/수정/삭제 가능 (`.env`나 별도 `shortcuts.json`에 저장)
- 자주 쓰는 검색을 숏컷으로 저장하는 **"즐겨찾기"** 버튼

### 💡 3-C. 이슈 인라인 액션 (중요)

**배경**: 검색 결과에서 이슈를 보고 상태 변경, 댓글 추가를 하려면 Agent 탭으로 이동해야 함

**개선 내용**:
- 검색 결과 이슈 카드에 **빠른 액션 버튼** 추가:
  - `[💬 댓글 추가]` — 인라인 텍스트 입력창 등장
  - `[🔄 상태 변경]` — 사용 가능한 Transition 드롭다운 표시
  - `[🔗 Jira에서 열기]` — Jira 원본 URL로 외부 링크

## 개선 영역 4: 유사 이슈 검색 고도화

### 💡 4-A. 캐시 구축 진행률 표시 (긴급)

**배경**: 수백 건 처리 시 수분간 빈 화면, 사용자는 오류인지 진행 중인지 알 수 없음

**개선 내용**:
- 서버 → 클라이언트 SSE로 진행 상황 스트리밍
- UI: `[=====>    ] 123/350건 처리 중 (35%) — 예상 남은 시간: 약 2분`
- 에러 발생 이슈는 건너뛰고 계속 진행 (현재와 동일), 완료 시 스킵 목록 표시

### 💡 4-B. 배치 AI검증 (편의)

**배경**: 미해결 이슈 10건이면 AI검증 버튼을 10번 클릭해야 함

**개선 내용**:
- **`[🤖 전체 AI검증]`** 버튼 추가 — 미해결 이슈 전체를 순차 검증
- 검증 완료 이슈부터 결과를 실시간 업데이트 (SSE 활용)
- 이미 캐시된 이슈는 즉시 표시

### 💡 4-C. 유사 이슈 → 처리 초안 자동 생성 (핵심 신규)

**배경**: 유사 이슈를 찾아도 처리 방법을 참고해서 직접 댓글을 작성해야 함

**개선 내용**:
- AI검증 결과 아래에 **`[✍ 처리 초안 작성]`** 버튼 추가
- 클릭 시 Gemini에게 "현재 이슈 + 유사 이슈 처리 방법"을 제공하여 댓글 초안 자동 생성
- 생성된 초안을 수정한 뒤 **`[Jira에 댓글 달기]`** 버튼으로 바로 등록
- 예상 효과: 반복적인 서비스요청 처리 시간 50% 이상 단축

### 💡 4-D. 이슈 타입 확장 (보조)

**배경**: 서비스요청관리, 변경관리 2종만 지원

**개선 내용**:
- 환경설정에서 **임베딩 대상 이슈 타입 선택** 가능 (체크박스)
- Jira에서 사용 가능한 이슈 타입 목록 자동 조회 후 표시

## 개선 영역 5: 전반적인 UX 개선

### 💡 5-A. 대시보드 홈 (신규)

**배경**: 첫 진입 시 바로 채팅 탭이 열려 전체 현황 파악이 어려움

**개선 내용**:
- 새 탭 **🏠 대시보드** 추가 (기본 화면)
- 표시 항목:
  - 내 미해결 이슈 건수 / 오늘 완료 건수
  - Gemini/Jira 연결 상태 요약
  - 유사 이슈 캐시 마지막 갱신 시간
  - 최근 AI검증 결과 요약

### 💡 5-B. 다크/라이트 모드 (보조)

**개선 내용**:
- CSS 변수 기반 테마 전환
- 설정을 `localStorage`에 저장하여 다음 접속 시 유지

### 💡 5-C. 에러 메시지 개선 (보조)

**개선 내용**:
- HTTP 401: `"Jira 토큰 인증 실패 — 환경설정에서 PAT Token을 갱신해주세요"`
- HTTP 429: `"Gemini API 할당량 초과 — 1분 후 다시 시도하거나 API Key를 확인해주세요"`
- 타임아웃: `"Jira 서버 응답 없음 — VPN/사내망 연결을 확인해주세요"`
