# 01. 현황 분석 — 사용자 UX 문제점

**작성일**: 2026-03-14

---

## 분석 방법

소스코드(`config_ui.py`, `lib/*.py`, `ui/*`) 전체를 검토하여 실사용자가 업무 환경에서 느낄 수 있는 **불편함과 한계**를 기능별로 정리합니다.

---

## 1. 환경설정 (⚙)

### 문제점

| # | 문제 | 심각도 | 근거 |
|---|------|--------|------|
| 1-1 | **첫 실행 시 안내 부족** — 어떤 키를 어디서 발급받는지 UI 내에 가이드 없음 | 🔴 높음 | settings.py에 필드 정의만 있고 도움말 없음 |
| 1-2 | **설정 후 자동 반영 안됨** — 저장 후 다른 탭으로 이동해야 실제 적용 확인 가능 | 🟡 중간 | API 호출이 매번 `.env` 재읽기 방식 |
| 1-3 | **Jira 연결 상태 확인 없음** — Gemini는 상태표시(🟢/🔴)가 있지만, Jira PAT는 저장해도 유효성 검증 없음 | 🔴 높음 | `api_gemini_check()`는 있지만 Jira 동등 기능 없음 |
| 1-4 | **토큰 만료 감지 불가** — PAT Token이 만료돼도 조회 실패 시 일반 에러만 표시 | 🟡 중간 | HTTP 401/403 분기 처리 미흡 |

---

## 2. Gemini 채팅 (🤖)

### 문제점

| # | 문제 | 심각도 | 근거 |
|---|------|--------|------|
| 2-1 | **대화 히스토리 휘발** — 브라우저 새로고침 시 이전 대화 전부 사라짐 | 🔴 높음 | 히스토리를 메모리(JS 변수)에만 보관, 비영속 |
| 2-2 | **스트리밍 미지원** — 긴 응답 시 수 초간 빈 화면, 사용자는 로딩 중인지 알 수 없음 | 🔴 높음 | `generateContent` 단건 호출, SSE/streaming 없음 |
| 2-3 | **Jira 컨텍스트 분리** — "SCM3-15200 이슈 요약해줘"처럼 채팅에서 Jira 데이터 참조 불가 | 🟠 높음 | 채팅과 Jira 탭이 완전히 분리됨 |
| 2-4 | **대화 내보내기 없음** — 유용한 응답을 복사/저장하려면 수동으로 텍스트 선택 필요 | 🟡 낮음 | UI에 복사 버튼, 내보내기 기능 없음 |
| 2-5 | **시스템 프롬프트 고정** — 업무 맥락(내 담당 프로젝트, 역할) 주입 기능 없음 | 🟡 중간 | `generationConfig`에 systemInstruction 없음 |

## 3. Jira 테스트 (💬)

### 문제점

| # | 문제 | 심각도 | 근거 |
|---|------|--------|------|
| 3-1 | **JQL을 직접 써야 함** — 일반 사용자는 JQL 문법을 모름. 단 3개의 고정 숏컷만 제공 | 🔴 높음 | `SHORTCUTS` dict가 3개로 고정 (jira.py L13-17) |
| 3-2 | **검색 결과 조작 불가** — 결과 목록에서 이슈를 클릭해도 상세 내용만 보임, 직접 Jira 이동 불가 | 🟡 중간 | 이슈 key 링크가 Jira 외부 URL로 연결 안됨 |
| 3-3 | **페이지네이션 없음** — maxResults 30건으로 고정, 더 많은 이슈 조회 불가 | 🟡 중간 | `maxResults: 30` 하드코딩 (jira.py L161) |
| 3-4 | **검색 히스토리 없음** — 이전에 검색한 JQL을 다시 입력해야 함 | 🟡 낮음 | 검색창에 history 기능 없음 |
| 3-5 | **이슈 수정 불가** — 조회만 가능, 상태 변경/댓글 추가 등 Write 기능은 Agent 탭에만 분리됨 | 🟠 높음 | `jira_update_issue()`가 구현됐으나 Jira 탭에서 접근 불가 |

---

## 4. 유사 이슈 검색 (🔍)

### 문제점

| # | 문제 | 심각도 | 근거 |
|---|------|--------|------|
| 4-1 | **캐시 구축 시간이 매우 긺** — 이슈 수백 건을 Gemini Embedding API에 순차 호출, 완료까지 수분 소요 | 🔴 높음 | `_embed_text()` 루프 내 동기 호출 (embedding.py L120-142) |
| 4-2 | **진행 상황 표시 없음** — 캐시 구축 중 몇 건 처리 중인지 알 수 없음, 멈춘 것처럼 보임 | 🔴 높음 | 서버 측 진행률을 클라이언트에 전달하는 구조 없음 |
| 4-3 | **캐시 16MB 단일 파일** — `embedding_cache.json`이 16MB, 로딩·저장 시 I/O 부담 | 🟡 중간 | 모든 이슈 벡터를 단일 JSON에 보관 |
| 4-4 | **사용자 최대 3명 제한** — 나 + 2명 고정, 팀 단위 확장 불가 | 🟡 중간 | UI에서 사번 2개 추가 입력칸만 제공 |
| 4-5 | **이슈 타입 2종 고정** — 서비스요청관리, 변경관리만 지원 (embedding.py L15) | 🟡 중간 | `EMBED_TYPES` 하드코딩 |
| 4-6 | **AI검증 버튼 이슈별 수동 클릭** — 미해결 이슈가 10건이면 10번 클릭 필요 | 🟡 중간 | AI검증이 개별 이슈 단위로만 동작 |
| 4-7 | **유사도만 보여줌, 활용 연결 없음** — 유사 이슈 확인 후 처리 방법 복사, Jira에 댓글 달기 등 후속 액션 없음 | 🟠 높음 | 검색 결과 화면에 액션 버튼 없음 |

## 5. 전반적인 UX/시스템 문제

| # | 문제 | 심각도 |
|---|------|--------|
| 5-1 | **다크모드/라이트모드 없음** — 장시간 사용 시 눈이 피로함 | 🟡 낮음 |
| 5-2 | **모바일 대응 없음** — 긴급 상황에서 폰으로 접속 불가 (로컬 서버이므로 원천 불가) | ⚪ 해당없음 |
| 5-3 | **알림 기능 없음** — 미해결 이슈 증가 시 자동 알림 없음 | 🟡 중간 |
| 5-4 | **단일 사용자 고정** — 팀원과 공유하거나 서버에 배포하려면 코드 수정 필요 | 🟢 해결중 | Docker Admin 가이드 및 Client-side Key 로직 도입 |
| 5-5 | **에러 메시지 불친절** — 서버 에러 시 `"서버 오류: ..."` 수준의 메시지, 해결 방법 미제시 | 🟡 중간 |
| 5-6 | **자연어 Jira 검색 미완성** — Agent 기능이 구현됐지만 별도 탭으로 분리되어 사용성 낮음 | 🔴 높음 |
