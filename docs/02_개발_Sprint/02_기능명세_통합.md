# 02. 기능 명세 — 종합 시스템 파이프라인 (UI/UX 및 백엔드)

**마지막 수정일**: 2026-03-19

진행 중이거나 새롭게 설계된 환경설정, UI/UX 디자인 메커니즘, 그리고 백엔드의 핵심 AI 검색 로직 설계도를 규합한 통폐합 명세서입니다.

---

## 1. 프론트엔드 환경설정 및 UI/UX 개선

- **사이드바(SPA) 및 레이아웃 분리**: 각 탭에 중구난방 흩어져 있던 기능(Gemini, Jira 테스트, 유사이슈 등)과 환경설정 패널(API 키, 사번, 토큰)을 분리된 사이드바 앱 구조로 재조정합니다.
- **클라이언트 데이터 지속화**: 서버 부하와 인증 관리를 분리하기 위해, API Keys, 유저 정보, 현재 채팅 내역을 `localStorage`에 저장하여 Stateless 서버를 유지합니다.
- **실시간 프로그레스 바 (SSE 통신)**: 대용량 데이터 전송 및 AI 응답 처리 시간을 시각화하기 위해 text/event-stream 방식의 % 진행바, Toast 알럿, Dot-spin 로딩 애니메이션을 적용합니다.
- **AI검증/처리 초안 시각화**: 
  - 생성된 이유(Reasoning)나 추천 대응 초안(Drafts)을 단순 읽기 쓰기가 아닌 수정/검수가 가능하도록 `textarea` 폼에 제공.
  - 관련 있는 유사도 점수별로 스타일 분리 표기.

## 2. 백엔드 AI 기능/데이터 로직명세

### 💡 자연어 기반 "Agent Query"
- **개요**: 일반적인 한국어 명령어를 분석해 `SEARCH(검색)`, `ACTION(조작)`, `CHAT(기본 대화)`로 의도를 분류하고 필요한 API 파라미터(JQL 등)를 구조적으로 반환시킵니다.
- **설계 구조**: `/api/agent-query` 엔드포인트를 통해 Gemini Prompt를 경유하며, 액션에 도달하기 전 Preview-approve 구조를 통해 Human-in-the-loop 안전장치를 둡니다.

### 💡 Gemini 기반의 유사 이슈 임베딩 및 검색 기능
- **이슈 유사도 계산 (Embedding)**:
  - 지정한 담당자들의 수백 개 완료 티켓을 추출하고, Python의 코사인 유사도 연산으로 현재 처리 이슈와 실시간 비용 없이 비교 매치.
  - SQLite를 적용하여 파일 단위 JSON 캐시에서 오는 Lock/동시성 충돌을 근절.
- **AI 2차 검증 (Re-validation)**:
  - 1차 알고리즘 기반 Top-N 결과 중, 실제 업무 맥락(의사결정)과 가장 부합하는 티켓 딱 1건을 Gemini LLM 모델이 판단 및 추천하게 만듭니다.

## 3. 개발 서버 및 인프라
- **컨테이너 (Docker) 호환성**: 모든 팀 단위 사용이 용이하도록, 파이썬 기반 서버와 캐시 DB 구조를 경량 리눅스 이미지(slim) 기반 컨테이너 스크립트로 제공. 멀티 환경에서도 독립 스레드 배포 환경(`ThreadingHTTPServer` 반영)을 준수합니다.
